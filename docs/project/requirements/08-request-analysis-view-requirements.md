# 单次请求分析与统计展示需求

## 需求列表

- REQ-VIEW-001: 请求分析视图以用户选择的某一次请求为主视图。
- REQ-VIEW-002: 同类其它请求只作为统计摘要展示，不在主图中逐个展开。
- REQ-VIEW-003: 选中请求中的不同应用分开展示。
- REQ-VIEW-004: 跨应用业务阶段、应用间传递阶段、应用内部阶段使用可区分颜色展示。
- REQ-VIEW-005: 阶段展示名称使用规则集配置的阶段名称。
- REQ-VIEW-006: 用户点击某个阶段后，可以查看该阶段起止边界日志和上下文日志。
- REQ-VIEW-007: 统计摘要包含请求数、平均耗时、P90、异常请求数、最慢应用、最慢阶段。
- REQ-VIEW-008: 展示层只消费加工后的请求、阶段、统计和日志引用数据，不直接承担解析、识别或计算职责。

## 主流程

1. 用户在请求列表中选择某一次请求。
2. 系统展示请求总览和统计摘要。
3. 系统按应用分开展示该请求内日志和阶段。
4. 系统在图中展示应用处理、应用间传递和应用内部阶段。
5. 用户点击阶段。
6. 系统展示该阶段边界日志、上下文日志和阶段耗时。
7. 用户可回到搜索结果或请求列表继续定位。

## 展示原则

- 主图只服务当前选中请求。
- 统计用于对比和发现慢点，不替代原始日志证据。
- 应用前缀用于展示分组，但不承担业务流程推断。
- UI 后续改版不应导致日志解析、请求识别、时延计算大改。

## 验收标准

- 用户可以从请求列表进入某次请求的时延视图。
- 用户可以在图上区分应用处理、应用间传递和应用内部阶段。
- 用户点击阶段后可以看到阶段边界日志和上下文日志。
- 同类请求统计可以辅助判断当前请求是否偏慢。
- 常用请求切换和时延视图切换 P90 <= 1s。
