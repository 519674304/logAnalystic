# 范围与约束

## 范围内

- 本地选择 `.log` 和 `.txt` 日志文件。
- UTF-8 日志文件解析。
- 固定格式单行日志解析。
- 多文件按时间合并。
- 关键字、短语、正则、结构化字段过滤。
- 保存查询的本地持久化。
- 业务规则集的本地导入、编辑、导出、覆盖、备份、恢复。
- 根据规则识别请求。
- 根据规则计算请求总耗时、跨应用阶段耗时、应用内部阶段耗时、应用间传递耗时。
- 以单次请求为中心展示时延，提供统计摘要和原始日志钻取。

## 范围外

- 日志采集、拉取、上传、清洗、标准化、多格式适配。
- 自定义任意日志格式解析器。
- 自动生成业务规则。
- 错误原因知识库、自动诊断结论、修复建议。
- 查询结果、请求分析、统计数据导出。
- 登录、权限、租户、审计、多用户协作。
- 服务器化部署、远程仓库同步。

## 数据约束

- 输入日志文件仅支持 UTF-8。
- 输入日志文件后缀仅支持 `.log` 和 `.txt`。
- 单次加载总日志量目标上限为 30MB。
- 日志时间戳按当前固定格式解析；跨年、多时区、时间校准不在首批范围。
- 解析失败行不参与搜索和时延分析，只进入数据质量列表。
- 日志、解析结果、索引、请求识别结果和时延统计只保留当前会话；关闭软件后不保留。
- 规则集、上一版本规则备份、保存查询保留在本地。

## 性能与容量

| 场景 | 目标 |
| --- | --- |
| 5MB 以内日志加载到可查询 | P90 <= 1s |
| 30MB 以内日志加载到可查询 | P90 <= 5s |
| 常用搜索、过滤、请求切换、时延视图切换 | P90 <= 1s |
| 峰值内存 | <= 2GB |

加载期间，搜索和时延分析在全部日志解析、合并、分析完成后可用。加载中需要展示进度和统计信息。

## 可靠性

- 文件级导入校验采用批次原子策略：任一文件扩展名、编码或读取失败，整批导入失败，并展示失败文件和原因。
- 行级解析失败不阻断整批导入；失败行进入数据质量列表。
- 正则执行超时时，中止该次查询，保留查询条件，并给出友好提示，不展示不完整结果。
- 规则集导入必须先完整校验；校验失败不覆盖当前规则，也不更新备份。

## 安全与运维

当前面向本机测试或开发使用，用户数量少，不做服务器化安全和运维能力。首批仅要求：

- 不引入登录和权限。
- 不主动上传日志或规则数据。
- 不保留会话日志数据。
- 对导入失败、解析失败、规则校验失败、正则超时提供可理解提示。
