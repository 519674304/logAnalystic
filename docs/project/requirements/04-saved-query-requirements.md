# 保存查询需求

## 需求列表

- REQ-SAVED-001: 用户可以保存当前搜索条件。
- REQ-SAVED-002: 保存查询包含名称、描述、分组、标签、关键字或正则、大小写选项和全部过滤条件。
- REQ-SAVED-003: 用户可以查看、调用、编辑、删除保存查询。
- REQ-SAVED-004: 保存查询在软件关闭后仍保留。
- REQ-SAVED-005: 保存查询是测试人员本地个人数据，不受规则集导入、覆盖和恢复影响。

## 主流程

1. 用户在搜索页面形成一组查询条件。
2. 用户保存查询并填写名称、描述、分组和标签。
3. 系统保存查询。
4. 用户下次打开软件后仍可查看保存查询。
5. 用户调用保存查询，系统恢复对应搜索条件。
6. 用户可以编辑或删除保存查询。

## 验收标准

- 保存查询后可立即调用。
- 重启软件后保存查询仍存在。
- 调用保存查询后，搜索条件恢复完整。
- 导入、恢复或覆盖业务规则集不会修改保存查询。
