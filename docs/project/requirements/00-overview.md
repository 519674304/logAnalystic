# 需求总览

## 目的

日志分析软件用于帮助测试人员在本机导入固定格式日志后，快速搜索关键日志、复用常用查询，并在规则集支持下识别一次业务请求，查看该请求中不同应用、不同阶段的耗时，定位慢点和问题证据。

## 用户与干系人

- 测试人员：主要使用者。导入日志、搜索日志、选择请求、查看时延和原始日志证据。
- 开发人员：规则维护者。提供或维护请求识别、阶段识别、分支识别和时延计算所需的业务规则。

当前不需要登录、权限、多人协作或集中式服务器能力。

## 核心业务场景

1. 测试人员选择一个或多个 `.log` / `.txt` 文件。
2. 系统按固定日志格式解析并合并日志。
3. 测试人员用关键字、短语、正则和结构化条件搜索日志，并可保存常用查询。
4. 测试人员导入或维护业务规则集。
5. 系统根据规则识别请求列表。
6. 测试人员按业务流程、时间、结束结果、是否异常筛选请求列表，并选择某一次请求。
7. 系统展示该请求内跨应用业务阶段、应用内部阶段、应用间传递阶段的时延。
8. 测试人员点击阶段查看边界日志、上下文日志和统计摘要，形成定位依据。

## 交付阶段

### Delivery 1: 基础能力

用户可以导入固定格式日志，完成解析、搜索、上下文查看，并保存常用查询。该阶段不做查询结果导出。

### Delivery 2: 进阶能力

用户可以导入业务规则集，系统识别一次请求并计算时延。展示以“用户选择的某次请求”为中心，其它同类请求只作为统计数据展示。

### 暂不进入首批交付

- 日志采集、拉取、上传、清洗、标准化、多格式适配。
- 错误原因解释知识库。
- 自动问题诊断结论。
- 查询结果导出、请求分析导出、统计导出。
- 多用户认证、权限、集中式协作。
- 远程仓库同步或服务器部署。

## 成功标准

- 测试人员无需开发协助即可在本机完成日志导入、搜索、请求选择和时延查看。
- 对 5MB 以内日志，加载到可查询状态 P90 <= 1s。
- 对 30MB 以内日志，加载到可查询状态 P90 <= 5s。
- 常用查询、过滤、请求切换、时延视图切换 P90 <= 1s。
- 峰值内存不超过 2GB。
- UI 展示不要求一次展开所有请求；单次请求明细优先，其它请求用统计摘要表达。

## 术语

| 术语 | 说明 |
| --- | --- |
| 固定格式日志 | 已经由外部系统生成且格式稳定的单行日志。本项目首批不处理采集、清洗和多格式识别。 |
| 应用前缀 | 日志中的应用标识，例如 `A00010`。用于过滤和展示分组，不用于自动推断业务流程或阶段。 |
| 业务流程 | 规则集中定义的一类请求识别逻辑。首批可能只有一个业务流程，但需求保留按业务流程筛选。 |
| 一次请求 | 从全局开始日志开始，到结束日志或下一次开始日志之前的一段日志范围。 |
| 结束日志 | 请求完成标志，可能不存在。不存在时，请求分析到下一次开始日志之前。 |
| 跨应用业务阶段 | 请求在不同应用或业务节点之间推进的阶段链，例如 A 处理、A 到 B 传递、B 处理。 |
| 应用内部阶段 | 同一个应用内部由规则定义的细分阶段。 |
| 应用间传递阶段 | 一个应用完成到另一个应用开始之间的耗时阶段。 |
| 保存查询 | 测试人员本地保存的搜索条件集合，不受业务规则集导入和恢复影响。 |
| 规则集 | 用于识别业务流程、请求边界、阶段、分支和结束结果的本地业务配置。 |
