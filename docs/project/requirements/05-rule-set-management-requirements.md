# 业务规则集管理需求

## 需求列表

- REQ-RULESET-001: 开发人员提供初始业务规则集。
- REQ-RULESET-002: 测试人员可以导入、查看、编辑、导出本地规则集。
- REQ-RULESET-003: 规则集属于本地业务配置数据，不与软件版本强绑定。
- REQ-RULESET-004: 导入规则集采用整体覆盖，不做 ID 合并。
- REQ-RULESET-005: 导入覆盖前自动备份当前规则集。
- REQ-RULESET-006: 只保留上一版本规则备份。
- REQ-RULESET-007: 导入规则集必须先完整校验；任一格式、引用、阶段顺序、完整性错误都会拒绝整次导入。
- REQ-RULESET-008: 导入失败时不覆盖当前规则，也不更新备份。
- REQ-RULESET-009: 规则匹配条件支持固定关键字或正则。
- REQ-RULESET-010: 规则可以限定应用前缀或日志字段。
- REQ-RULESET-011: 同一层级多个阶段规则命中时，使用最高配置优先级。
- REQ-RULESET-012: 同一层级同优先级冲突属于规则校验错误。
- REQ-RULESET-013: 跨应用业务阶段和应用内部阶段属于不同层级，可以复用同一条日志。

## 规则集需要表达的业务内容

- 业务流程定义。
- 全局开始日志规则。
- 可选结束日志规则和结束结果定义。
- 请求内阶段定义。
- 阶段类型：应用处理、应用间传递、应用内部。
- 阶段名称。
- 阶段开始日志匹配规则。
- 阶段结束日志匹配规则。
- 所属应用，适用于应用处理和应用内部阶段。
- 所属分支，适用于有分支的业务流程。
- 分支入口日志规则。
- 规则优先级。

## 主流程

1. 用户导入规则集。
2. 系统完整校验规则集。
3. 校验通过时，系统备份当前规则集。
4. 系统用新规则集整体覆盖当前规则集。
5. 用户可以查看、编辑、导出当前规则集。
6. 用户可以恢复上一版本规则备份。

## 验收标准

- 合法规则集导入后立即参与后续请求识别和时延分析。
- 非法规则集不会覆盖当前规则集。
- 覆盖前当前规则集被备份，且仅保留上一版本。
- 保存查询不受规则集导入、覆盖、恢复影响。
